클로드 코드로 마크다운 블로그를 직접 만들어 보면서, 웹페이지를 이루는 세 가지 기술이 각각 어떤 일을 맡는지 확실히 알게 되었다. 한 줄로 요약하면 이렇다.

> HTML은 **뼈대**, CSS는 **옷**, JavaScript는 **움직임**이다.

## HTML — 구조와 내용

HTML은 페이지에 *무엇이* 있는지를 정의한다. 제목, 문단, 목록, 버튼 같은 요소를 의미에 맞는 태그로 적어 두면, 그것이 페이지의 뼈대가 된다.

- `<header>`, `<main>`, `<footer>` 로 영역을 나눈다
- `<article>` 로 글 하나를 감싼다
- `<a>` 로 다른 페이지로 이동한다

내용과 구조만 담당하고, 어떻게 보일지는 신경 쓰지 않는다는 점이 핵심이었다.

## CSS — 디자인과 레이아웃

CSS는 그 뼈대가 *어떻게 보일지*를 결정한다. 색, 여백, 글꼴, 화면 크기별 배치가 모두 여기서 정해진다.

이번 블로그에서 특히 유용했던 건 **커스텀 프로퍼티(변수)** 였다. `--bg`, `--text` 같은 변수에 색을 모아 두니, 다크 모드는 변수 값만 바꿔서 구현할 수 있었다.

```css
:root {
  --bg: #ffffff;
  --text: #1f2328;
}

[data-theme="dark"] {
  --bg: #0f1115;
  --text: #e6e8eb;
}
```

모바일 대응도 CSS의 몫이다. 기본은 좁은 화면 기준으로 작성하고, `@media` 로 넓은 화면을 보정했다.

## JavaScript — 동작과 상호작용

JavaScript는 페이지가 *어떻게 움직일지*를 맡는다. 정적인 HTML/CSS만으로는 못 하는 일들이다.

1. `fetch` 로 마크다운 파일과 글 목록을 불러온다
2. 불러온 텍스트를 HTML로 변환해 화면에 그린다
3. 테마 토글 버튼을 누르면 테마를 바꾸고 `localStorage` 에 저장한다

버튼 클릭, 데이터 로딩, 화면 갱신처럼 "사용자와 주고받는 모든 것"이 JavaScript의 영역이었다.

## 정리

셋의 역할이 깔끔하게 나뉜다는 걸 몸으로 익혔다. HTML로 뼈대를 세우고, CSS로 보기 좋게 입히고, JavaScript로 살아 움직이게 한다. 프레임워크 없이 만들어 보니 각 기술이 맡은 경계가 오히려 더 또렷하게 보였다.
