# CLAUDE.md

마크다운 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다. 이 문서는 Claude Code가 이 저장소에서 작업할 때 따라야 할 규칙과 구조를 정의합니다.

## 핵심 제약 (반드시 지킬 것)

- **프레임워크 금지.** React, Vue, Svelte, Next.js 등 어떤 프런트엔드 프레임워크도 사용하지 않습니다. 순수 **HTML + CSS + JavaScript(ES 모듈)**로만 구현합니다.
- **빌드 단계 없음.** 번들러(Webpack, Vite 등)나 트랜스파일러를 도입하지 않습니다. 브라우저가 파일을 그대로 실행할 수 있어야 합니다.
- **의존성 최소화.** 마크다운 파싱은 `js/markdown.js`에 직접 구현한 경량 파서를 사용합니다. 외부 라이브러리(예: marked.js)를 추가하려면 먼저 사용자에게 확인을 받습니다.
- **클라이언트 사이드 렌더링.** 글은 런타임에 `fetch`로 불러와 DOM에 렌더링합니다. 서버 코드는 없습니다.

## 디자인 원칙

- **읽기 좋은 본문이 최우선.** 본문 컬럼은 최대 ~680px 폭, 넉넉한 `line-height`(1.6~1.8), 충분한 문단 간격을 유지합니다.
- **다크 모드 지원.** CSS 커스텀 프로퍼티(`--bg`, `--text`, `--accent` 등)로 테마를 정의하고 `[data-theme="dark"]`로 전환합니다. 최초 진입 시 `prefers-color-scheme`를 따르고, 사용자가 토글하면 `localStorage`에 저장해 유지합니다.
- **모바일 우선 반응형.** 모바일 레이아웃을 기본으로 작성하고 `min-width` 미디어 쿼리로 넓은 화면을 확장합니다. 터치 타깃은 최소 44px, 가로 스크롤이 생기지 않게 합니다.
- **깔끔하고 절제된 UI.** 장식보다 여백·타이포그래피·대비로 위계를 만듭니다. 색상은 테마 변수에서만 가져오고 컴포넌트에 하드코딩하지 않습니다.

## 디렉토리 구조

```
my-blog/
├── CLAUDE.md
├── index.html          # 홈 — 글 목록
├── post.html           # 개별 글 페이지 (?slug=... 로 글 지정)
├── css/
│   └── style.css       # 테마 변수, 타이포그래피, 반응형 레이아웃
├── js/
│   ├── app.js          # 진입점 — 목록/글 렌더링, 라우팅
│   ├── markdown.js     # 마크다운 → HTML 파서 (자체 구현)
│   └── theme.js        # 다크/라이트 토글 + localStorage 유지
├── posts/
│   ├── index.json      # 글 메타데이터 목록 (브라우저는 폴더를 못 읽으므로 필수)
│   └── *.md            # 마크다운 글 본문
└── assets/             # 이미지 등 정적 파일
```

### 글 추가 흐름

1. `posts/`에 `my-post.md` 파일을 추가합니다.
2. `posts/index.json`에 항목을 추가합니다: `{ "slug", "title", "date", "summary" }`.
3. 목록과 개별 페이지는 `index.json`을 읽어 자동으로 채워집니다. **`index.json`을 갱신하지 않으면 글이 노출되지 않습니다.**

마크다운 파일 상단의 frontmatter(`---`로 감싼 메타데이터)를 지원할 경우, `markdown.js`에서 파싱해 `index.json`의 메타데이터와 일관되게 다룹니다.

## 로컬 실행

`fetch`는 `file://`에서 동작하지 않으므로 **반드시 로컬 HTTP 서버로 실행**합니다.

```powershell
# Python이 있는 경우
python -m http.server 8000

# 또는 Node가 있는 경우
npx serve
```

이후 브라우저에서 `http://localhost:8000` 으로 접속합니다.

## 코드 컨벤션

- JavaScript는 ES 모듈(`<script type="module">`)과 `import`/`export`를 사용합니다.
- DOM 조작은 `textContent`를 기본으로 쓰고, 마크다운 변환 결과처럼 HTML 삽입이 필요한 곳에서만 `innerHTML`을 사용합니다. 사용자 입력을 그대로 `innerHTML`에 넣지 않습니다.
- CSS는 `style.css` 한 곳에서 관리하고, 색·간격·폰트는 `:root`의 커스텀 프로퍼티로 정의해 재사용합니다.
- 함수와 변수 이름은 명확하게, 불필요한 주석 대신 읽히는 코드를 우선합니다.

## 작업 시 주의

- 새 글이나 기능을 추가한 뒤에는 로컬 서버로 띄워 **목록 페이지 → 개별 글 페이지 → 다크/라이트 토글 → 모바일 폭**까지 실제로 확인합니다.
- 위 "핵심 제약"을 벗어나는 변경(프레임워크·빌드 도구·외부 라이브러리 도입)은 진행 전에 사용자에게 먼저 묻습니다.
